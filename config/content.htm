為何需要以子模組的設計

在一個倉儲中帶有子模組的用意是善用分散式設計的做法, 子模組代表一個非由倉儲管理者維護權限所及的單位模組. 意即, 倉儲中程式的執行需要子模組中的資料, 但是倉儲擁有者將子模組視為獨立元件, 並不需要對子模組的內容進行改版. 一旦子模組原始倉儲資料更新, 採用子模組的各倉儲所有者可以決定是否配合跟進, 若不跟進仍有使用既有版本的子模組繼續運作, 原始子模組與採用不同版本子模組的倉儲都不會彼此受到影響.

PCS 14 Prevention Methods

overview

In this module, we will discuss security, including common physical and digital prevention methods. When exploring digital security, we will take a deeper dive into user authentication, such as passwords. We will also discuss securing wireless and wired networks.

Physical Security

While most people think of passwords, encryption, and firewalls as the best methods to protect a computer, they often forget that computer security begins with physical security. While this may sound quaint or silly, it is often the easiest method for an attacker to gain access to a system. After all, what good does a firewall do in protecting your computer from outside attack when the attacker is physically at the computer?

It is important to ensure that doors are locked so as to ensure the attackers are kept out of the building. Server rooms and communication closets must be kept under lock and key, as these are target-rich environments for attackers. Door locks come in a variety of styles, including the traditional key lock, while some organizations prefer more high-tech physical security devices such as digital numeric keypad locks, radio frequency identification (RFID) card systems, or even biometrics for entry, such as fingerprint readers or retina scanners.

Not only should the server rooms and communication closets be kept locked, but the main entrance to the facility should be secured, as well. If your organization restricts access to entry points, it is important that you be mindful of tailgating. Tailgating is the practice that occurs when another person tries to enter the facility immediately behind you, effectively using your card swipe or authentication to enter the facility. Many secure facilities use turnstile systems or mantraps to allow a single employee to enter upon swiping a card key or access badge, or entering a personal identification number. A mantrap is designed to prevent tailgating.

While physical access can be thwarted by locking doors and preventing tailgating, employees should also be mindful of protecting the physical data that exists in their workplace environment. To prevent work-desk disclosures, you should secure documents in locked file cabinets at the end of the day, shred any sensitive documents at the end of your shift, and refrain from writing down passwords on physical pieces of paper. Another good method to prevent prying eyes from seeing the documents with which you are working on the computer is to utilize a privacy filter over your monitor.

There are many methods of providing authentication to enter a facility, such as biometrics, key fobs, RFID, and smart cards. Biometric security systems utilize a reader or scanner to analyze the employee’s physical characteristics. For example, if a fingerprint reader is utilized, the reader can find a series of match points from the employee’s fingerprint, then compare them to a database of approved users. For retinal scans, the scanner reads match points in an employee’s retina. Some laptops and mobile devices now incorporate biometric data such as fingerprint readers in place of a username and password. Biometrics are more secure than a traditional username and password, but they are also more intrusive to the employee. Biometrics can also be used to authenticate you by certain unique traits or patterns you have such as comparing baseline models of your signature, voiceprint, heartbeat rhythm, and even keystroke measurements.

Other security identification methods are key fobs and RFID tags. Key fobs utilize a timing algorithm that is in sync with the global positioning system (GPS) satellites as a time source. The key fob then provides the user with a six-digit code that changes every 15–30 seconds. If the code is not entered within this time period, it becomes invalid and a new code is provided. This code is unique to the user and provides a unique method of authentication. RFID tags can be located in a key fob, access badge, or other device. These devices contain information about the identity of the user, which is passed to a reader using radio frequencies. These RFID tags are often combined with a security badge and a personal identification number (PIN), providing secure, two-factor authentication for the employee at physical access points, such as turnstiles and mantraps.

Another method of physical identification is the smart card. Smart cards are used by Department of Defense personnel. The cards act both as identification cards, much like driver’s licenses, and as login credentials for the users’ computers. A smart card contains a small nanoprocessor embedded in the card that contains digital certificates for email signing and encrypting functions. The card is placed into a reader connected to the computer via USB and is unlocked by providing the user’s personal identification number. Use of the smart card provides secure, two-factor authentication by using both the card and the PIN to unlock it.

Sample smart card. Image used under CC-BY-SA license from Wikimedia.

Password Security

Security is increasingly a concern in our world, and requires controlling both physical and virtual access. Computer and data protection are vital to businesses large and small, and on a personal level as well. The industry model of authentication, authorization, and accounting (AAA) has evolved from two-factor authentication to multi-factor authentication. When a user provides simply a username and password, this is considered single-factor authentication. In modern systems, as the need for security increases, so does the need for multiple factors of authentication.

The authentication measures employers require of employees ensure protections are verified through personal identification. Think of authentication levels as:

  1. Something you know: a username, password, pin, or another type of passcode or phrase.
  2. Something you have: a Smart Card, RFID tag, or some other token provided by the employer.
  3. Something you are: Biometric access through voice, fingerprint, or retinal identification.
  4. Something you do: another form of Biometrics using signature pattern analysis or voiceprint analysis.

“Something the user knows” is the easiest authentication for an attacker to overcome, because the attacker simply needs to guess or collect the user’s knowledge to gain access. “Something the user has” may be more difficult for an attacker to acquire, but not impossible. “Something the user is" should be the most difficult authentication measure to forge, but again, it is not impossible to do so. While movies contain many graphic representations on how to achieve this, there are real technological hacks available that do not require a body part. Biometric sensors can be fooled. “Something the user does” brings biometrics to a new level and is not very common except in the most secure environments. In addition to what somebody is, this authentication creates a baseline for comparative analysis of the person’s known patterns. Signature and voiceprint analysis, keyboards that identify the user by keystroke pressure and typing rate, and even physical measurements of heart rate, body heat, and blood pressure are now being used to complete this fourth type of authentication.

To ensure a high level of security, many workstations and websites now require a minimum of two-factor authentication. As we previously mentioned, users accessing the Department of Defense computer systems are required to insert their Common Access Card (CAC), which is a Smart Card, and then enter a personal identification number to gain access to the system. This two-factor authentication provides both something the user has (the smart card) and something the user knows (the PIN) to authenticate the user and log the user onto the workstation.

PCS 14 /Digital Security

In addition to the physical security, the computer must also be protected through digital security, as well. This takes many forms, including using antivirus/antispyware/antimalware software, firewalls, proper user authentication, securing the wireless network, and securing the rest of the network.

Antivirus, Antispyware, and Antimalware Software

Whether you are in a business or home computer environment, every device (tablet, cell phone, or workstation) should have antivirus, antispyware, and antimalware software installed. Most antivirus software now includes scanners for all three. This is specialized software that starts at boot-up and runs in the background of the machine protecting the device in real time from infections. By default, antivirus software will scan the machine when it boots to ensure there are no boot viruses or rootkits that will affect the startup process. In addition, the software should be scheduled to automatically run a full scan at periodic intervals for known and suspected threats.

The software must be kept up to date, as new viruses and malware are created all the time. In addition to keeping the program updated, the "signature" file, which tracks the latest threats, should check for updates daily. Most antivirus programs operate on a subscription basis, where the end user must pay an annual fee to keep the software up to date. However, there are excellent freeware programs available from third-party vendors. Additionally, this software provides access to the virus/malware threat encyclopedia, where the user can become more educated about a particular threat that exists.

Running antivirus/antimalware software protects the system against software threats that can affect not only that device, but the entire network. Often, antivirus software is offered as part of a suite of products including a firewall, browser protection, password management, and family controls that allow permissions-based access to the Internet. Good antivirus software will also integrate into the user’s locally installed email program, ensuring scanning of all downloaded files and sent/received emails.

Since Windows Vista, Microsoft has included Windows Defender as an installed product that can run concurrently with third-party vendors. From Windows 8 on, it is an either/or proposition, where Defender will be disabled once you install another antivirus product.

Firewalls

Firewalls are either hardware devices or software programs that are designed to examine data packets that traverse the network connection. This includes IP header information, such as the destination or source IP addresses, the application ports and data being utilized, or the protocols being used. Based on this information, the firewall makes the determination to either allow or block the particular network traffic. In Windows Vista (by configuration) and from Windows 7 on (by default), there is a two-way firewall solution. In addition to running locally on each machine, the Windows software-based firewall can be managed by a Windows server.

Hardware firewalls are dedicated network appliances with specially designed operating systems that are more efficient at protecting multiple devices in the network. Often, networks utilize a hardware firewall at the network boundary to protect the entire network from a broad range of attacks, while software firewalls are configured to protect specific workstations from specifically targeted attacks. A technician should never install more than one software firewall on a workstation, as two firewalls can conflict and block valid network traffic from flowing properly. Also, note that Windows comes with its own software firewalls integrated into the operating system, but they can be turned off if a third-party, more in-depth solution is desired.

Firewalls can block legitimate network traffic if they are misconfigured. It is important that the technician check the firewall settings if a program appears to not be working properly to ensure the correct TCP or UDP ports are opened in the firewall. For example, if a user complains that their email program is unable to send emails, it may be that the firewall is configured to block traffic over port 25 (the SMTP — simple mail transfer protocol — server).

Least Privilege

A common security best practice is that of “least privilege.” This principle dictates that a user should only be given access to the minimum permissions and groups that they need to perform their job functions. For example, if the user is a data entry clerk, he may need to be able to update spreadsheet files and browse the Internet, therefore he should be given standard user access, not administrator access. This may seem like common sense, but it can become difficult to manage permissions once you begin working on a large-scale network. It is imperative that the administrator who is creating the user accounts and placing them into the groups take extreme care to place the users in the proper groups.

As an administrator, be aware that when you install a program under Windows Vista or Windows 7, the program often will ask which user groups should have access to the program, with the default option often being “all users.” This may not be the correct option, especially if the software is something you only wish for the administrators to access. Remember, organizations use groups for classifying users into need-to-know or need-to-access groups. So giving users additional, unnecessary groups can allow the users to access information that management does not wish certain people to access.

PCS 14 /User Authentication

User authentication typically relies on the user providing a username and password, but in modern networks the need for security is changing this requirement. When a user provides simply a username and password, this is considered single-factor authentication. In modern systems, as the need for security increases, so does the need for multiple factors of authentication, such as two-factor authentication (at a minimum) or multi-factor authentication.

There are four common categories of authentication:

  1. Something the user knows.
  2. Something the user has.
  3. Something the user is.
  4. Something the user does.

Something the user knows is typically used by most systems, such as the user providing a username, password, or personal identification number (PIN). This is also the easiest authentication for an attacker to overcome, since they simply need to guess or collect the user’s knowledge to gain access. The second category is something the user has, which is usually fulfilled by the user presenting a smart card, security token, or RFID chip. The third category is something the user is, which usually refers to some biometric data about the user. This could be the user’s fingerprint, retina scan, or even voice authentication. The fourth and final category is something the user does, such as how the user’s signature looks. This is a very uncommon authentication method for workstation or website access, but is used heavily in payment systems, such as credit card terminals.

To ensure a high level of security, many workstations and websites now require two-factor authentication. As mentioned earlier in this module, in the Department of Defense computer systems, the user is required to insert their CAC (common access card), which is a smart card, and also enter a personal identification number to gain access to the system. This two-factor authentication provides both something the user has (the smart card) and something the user knows (the PIN) to authenticate the user and log the user on to a Windows workstation.

Passwords

Because usernames and passwords are the most common method of authentication, we will dig a little deeper into the best practices for a password system. Passwords can be verified by the local system, such as by using a username/password with a local PC or an access code number for a digital door lock. Passwords can also be verified by using a remote server, such as when a domain user logs in to a workstation connected to a domain controller. The local system does not verify the password; the domain controller does.

The threat of security breaches has changed the whole concept of passwords from the days when they were simply used to keep another user from accessing your profile. Data has become an important form of currency throughout the world, so the need for password complexity has evolved. The first rule of data security, which most users will admit to breaking sometimes, is to never write down a password on paper and leave it in the desk or on a sticky note on the desk or monitor. Passwords need to be memorized or stored in a password manager utility where access is controlled by a single password. The second rule, which is also often broken, is to never use the same password for multiple different things. Some of the most recents hacks have involved two or more databases from online vendors, and the attackers were able to merge the information to develop usernames and passwords. Those using the same password for multiple accounts were most vulnerable.

Passwords can be verified by the local system, such as by using a username/password on a local PC or an access code number for a digital door lock. Passwords can also be verified by using a remote server, such as when a domain user logs in to a workstation connected to a domain controller. The local system does not verify the password; the domain controller does.

Passwords must be complex, containing a mixture of uppercase and lowercase letters, numbers, and symbols. Additionally, passwords should be long: they should be at least 6–8 characters in length, but more characters provide a more secure password. It is imperative to avoid dictionary words, as these are easier to guess, and consider using a passphrase if possible. One technique to create a strong password is to pick a song or poem and use the first letter from each word as the password, as well as incorporating some numbers and symbols. This makes a strong, hard-to-guess password.

The system administrator has the ability to configure the password system in Windows to create rules that the passwords must follow. These settings should be in line with company policy and should follow the best practices mentioned above, as well as adding some new requirements. One practice is requiring passwords to be renewed every 60 days. Setting a maximum password age eliminates threats from old access lists or former employees. The system can be set up to prompt users to change their passwords approximately 14 days prior to expiration. Also, the system administrator can set the system to require a minimum password length, such as six characters or 14 characters. There is an option to require complex passwords — passwords that require uppercase, lowercase, numeric, and symbolic characters. Another option is to prevent old passwords from being reused, with a setting that can dictate how many old passwords are remembered by the system. This is called password history. The system administrator can set the limit on password history up to 25 previous passwords in memory. Finally, if an attacker is trying to guess a user’s password, there is a setting that will prevent this by locking the user’s account if a certain number of unsuccessful logins occur within a certain period of time. For example, if four failed logins occur within 15 minutes, the system might require a 60-minute wait before letting the user log in (even if the user tries the correct password) or might require the system administrator to reset the password.

PCS 14 /Education

In addition to securing a wireless network, it is also important to secure a wired network. Just like the wireless network, the wired switch can be configured to support MAC address filtering. Additionally, any unused network ports should be turned off, as this will prevent a local attacker from connecting to an available port to use in their attacks. Also, cover any unused network jacks. For example, you would not want an open and available network jack in the waiting area of your business to be used by a would-be attacker. Finally, consider if your network should use static or dynamic addressing.

Static IP addresses are more secure, because they require an attacker to know the manual configuration for your network to gain access, but it is more time consuming for administrators, especially if your organization has a very large network. Static IPs should be used at a minimum for servers and devices that must be regularly contacted for their services, such as shared scanners and printers. If your organization uses dynamic addresses, it is a good practice to limit the scope of the DHCP server. This will allow clients to gain access to the network, but will minimize the number of overall IPs available.

EXAMPLE

If your company has 50 clients, then you should only allow 50 DHCP connections. If an attacker tries to connect to your network, she won’t be given an IP address because your 50 available connections are already used by the company’s 50 clients.

User Education

User education is the most important thing an organization can do to protect its network. If your users do not understand security threats, they will be unable to help your administrators fight the attackers. The employees must be made aware of any company policy for computer use, data protection, and allowed devices. Signing a form at the initial hire is not sufficient, and it is the IT administrator's duty to ensure that periodic training is completed. If it is company policy that users must keep their workstations up to date with the latest security patches, antivirus signatures, antimalware signatures, and antispyware signatures, then system administrators must assist with that process. Most IT departments handle patches and updates for their users, and the users are only responsible for antivirus scanning or ensuring that they don't tamper with the automated scanning schedule.

Administrators can use group policy to audit users' activity on the network and can require remediation, counseling, or training if the company policy is violated. Administrative tools are available to suspend users' access in the event that they violate policy or put the network at risk. Administrators should disable auto run on users' machines and should ensure that users understand that they should not install unapproved programs or insert removable media into the workstation without prior antivirus scanning. Bring your own device (BYOD) policies need to be clear so that employees understand what is allowed and what is not. Celluar telephones may be allowed on the network, but a wireless access point should not. The weakest area in most organizations is email access. Training employees how to spot potential spam or malicious attachments is essential.

The more you can educate users about risk and prevention, the more users will be prevented from falling for attackers’ tricks, such as phishing emails, and the more secure your network will be. All the firewalls and security software available will not do any good to protect your network if a user gives the attacker access and allows the attacker to bypass your defenses.

PCS 15 /Threats and Practices

Overview

In this module, we will explore different types of technical and nontechnical security threats. We will also discuss threats such as Trojan horses, which combine technical and nontechnical attack techniques. Additionally, we will learn about strategies to help prevent attacks.

Social Engineering

Social engineering is a major security threat. Social engineering refers to the use of a variety of techniques to convince computer users to provide access to their systems. This was popularized by computer and network hacker Kevin Mitnick, who said, “People are prone to taking mental shortcuts. They may know that they shouldn't give out certain information, but the fear of not being nice, the fear of appearing ignorant, the fear of a perceived authority figure — all these are triggers, which can be used by a social engineer to convince a person to override established security procedures.” For example, if a hacker walks up to the secured area of an office building dressed as a package delivery man with his hands full, how many people do you think would open the door to let him in? This type of deception is at the heart of social engineering.

Pretexting

Pretexting is a form of social engineering. Pretexting occurs when a social engineer presents some limited information to an authorized user to gain access to her computer system or gain access to the business.

EXAMPLE

A receptionist receives a phone call and the caller says, “Hello, my name is Bob from the IT help desk. We have detected that your computer password has been compromised and needs to be changed. First, can you verify your name and date of birth for me?” At this point, the receptionist provides those details, which can be used to gain access to other accounts. As the call continues, the attacker asks for the receptionist’s username and password, to “verify” that the account has been corrupted. In the end, the receptionist’s account has been compromised by the caller.

Phishing

Phishing is another form of social engineering. Phishing attacks are an effective and easy way for an attacker to gain access to a system or gain information about a user’s account or financial data. Traditionally, phishing involves the end user receiving an email stating his account has been compromised, or another similar premise. In the email, there is a link to click to reset the password or to log in to his account to verify data.

When the user clicks the link, he is taken to a website that may look like his bank or other service, but the address may be slightly incorrect. For example, instead of taking the user to Facebook.com, it may be Facebooks.com. When the user enters his username and password, the user is then forwarded to the correct site, but the attackers can now use his compromised credentials in future attacks. In the case of a bank or brokerage account, the attackers can now access that account and transfer the money in the accounts to themselves or collect other details in order to monetize the exploit.

Baiting

Baiting occurs when an attacker leaves physical media — such as CDs, DVDs, or USB thumb drives — lying around for a victim to find. Once the victim finds the media, she is curious to find out what is on it (either to learn who the owner is to return it, or to see the confidential data enclosed on the media). Often, these tainted devices could be presented to a victim in a myriad of seemingly legitimate ways, such as a free gift or as a software update. The more enticing the device, such as a smartphone being left around, or the more believable the scenario under which the victim obtains the device (such as a free raffle), the more difficult it is for the victim to recognize the situation as a potential baiting scheme. The best way to prevent a baiting attack is to ensure employees are aware they exist, so they can resist the temptation.

Shoulder Surfing

Shoulder surfing occurs when an attacker attempts to gain access to confidential information by looking over a user’s “shoulder” at the user’s screen, keypad, or keyboard. An example is someone looking over your shoulder at the ATM to see your PIN as you type it on the keypad. Another example might be if you are looking at the screen of a billing clerk at a business. The clerk may still have another person’s account (and financial information) up on the screen. Some shoulder surfers act covertly by using mirrors, peeking around corners, or even having hidden video cameras to capture the detailed information they seek. The best defense against this type of attack is to be aware of your surroundings and ensure people remain at a safe visual distance from your terminal. To help prevent shoulder surfing on your laptop and mobile devices, you can also use privacy screen protectors. These special screen protectors prevent screen viewing at an angle, making it difficult for anyone other than the user, who is looking directly at the screen, to see the display.

Tailgating and Piggybacking

Tailgating occurs when an attacker attempts to gain access to a secure facility by following closely behind an authorized person without her consent. If this access is gained with the consent of the authorized person, this is called piggybacking. A great example of this may be when a delivery person, like a pizza delivery or flower delivery person, asks for another employee to hold the door (after the authorized person swiped an ID badge), because the delivery person’s hands are too full of the product being delivered. By doing this, the “delivery person” is able to gain access to the building without authorization. This attack relies on the inherent kindness most people have, and it is hard to teach employees to forgo this common courtesy.

PCS 15 /Malware

Social engineering is a very effective way to gain access to a system without relying exclusively on the technical realm. In addition to social engineering, an attacker may also engineer a technical attack through malware. Notably, an attack combining social engineering with malware can be a significant threat.

The term malware is a combination of the words malicious and software, and it is used to describe any software used to disrupt or gain unauthorized access to a computer or network. There are numerous types of malware. In this lesson, we will discuss viruses, worms, Trojan horses, rootkits, spyware, remote access, adware, grayware, and spam.

Viruses

Viruses are software programs that infect files in an operating system, wreaking havoc on the system and doing what an attacker wishes. Viruses require replication to spread, and this can only be accomplished when a user executes an infected program containing the virus. Worms are similar to viruses, but they self-replicate across a network. Worms are a very effective type of malware, because they can self-replicate without any user intervention. This also makes worms very dangerous. Trojan horses are programs that claim to be useful utilities but actually install harmful programs on your computer, including spyware, remote access, and rootkits. Trojan horses are usually combined with social engineering, such as phishing, and are used as a delivery mechanism for more serious threats to your computer and network.

Rootkits

Rootkits are a concealment method used by many types of malware to prevent detection by normal antivirus and antimalware programs. Rootkits tend to overwrite and rename system files, replacing these necessary files with themselves. This embeds the malware into the operating system, making it difficult to detect through standard antivirus software and instead requires advanced detection techniques outside of the normal operating system to detect and remove this variation of malware. Rootkits are activated each time a system boots up.

Spyware

Spyware is software that spies on system activities and transmits details of web searches or other activities to remote computers. A good indication that spyware has infected your computer is if you get multiple pop-up windows when browsing the Internet. Some of these pop-ups may offer you the ability to remove a virus or infection, but you should never use them. These are scams and usually are rogue or fake antivirus/antimalware products that often lead to a deeper infection across your computers and networks. Spyware also can slow down the performance of your computer, wasting valuable computing resources through bloated software code and spying activities.

Remote Access

Remote access malware programs enable unauthorized control of your system and can be used to set up networks of compromised computers. These malware programs are also known as botnets. Through remote access, attackers can control your computer, capture your keystrokes, turn on your webcam, and copy your files to their own machine, gaining access to your confidential records and accounts.

Adware, Grayware, and Spam

Adware is software that displays pop-up ads and banners related to your web searches and activities. Grayware is a generic term for dialers, joke programs, adware, and spyware programs. Spam is the unwanted receipt of emails from unknown entities, very similar to junk mail received in a physical form at your house. Spam can be malicious (such as trying to send you a virus), a scam (such as trying to get you to send money), or simply ridiculous (such as sending you jokes or other messages you didn’t desire).

Spam is usually sent by an attacker hijacking a user’s email account by cracking their password, then using that account to send out the mass amounts of spam. For this reason, it is important to frequently change your email account passwords, to prevent a spammer from cracking your password and using your account.

PCS 15 /Security Awareness

The adage “an ounce of prevention is worth a pound of cure” is still true in this modern computer age. One of the most effective methods of protecting your network and preventing malware infection is to ensure your users are receiving appropriate cybersecurity education. Preventing attacks and infections will save you a good deal of time and resources that would otherwise have to be spent troubleshooting an infection.

Train your users on methods to keep antimalware software (like antivirus and antispyware) up to date both at work and at home. Users need to understand how to scan their systems for malware and configure their applications for automatic scanning. A basic understanding of the various malware types/techniques is also necessary, so a user can identify the symptoms of an infected system. Everyone should scan removable media for malware and prevent auto-run from occurring on their systems, as this is a common intrusion vector for attackers. If a computer becomes infected, users should be aware of your organization’s policy on how to respond to malware threats, how to quarantine suspected files, and how to prevent the spread of the infection.

Additionally, a good anti-phishing education is essential, as this is one of the most common vectors of advanced persistent threats (APTs) occurring today. Anti-phishing education will help prevent users from being socially engineered to unwittingly opening the door to an attack, such as clicking on a Trojan attached to a suspicious email.

Physical Security

The text in this section is used under CC-BY license from Progressive, Accelerated Certifications for Employment in Information Technology (PACE-IT) TAACCCT Grant, retrieved from www.skillscommons.org.

Physical security is key to protecting your computers, systems and data from attack.

Security at the Building Level

Security begins at the door. Actually, it begins at every door. Locking doors is the first level of security. The boundaries of your building should be your first line of defense. The level of security that gets deployed should be driven by the amount of security needed. As the need for overall security increases, so should the level of physical security.

While it may not be practical to lock every door, there are certain doors that should definitely be locked. The doors to equipment or server rooms, where servers are kept, should be locked. The doors to network cabinets should also be locked, even if they're in the server room. In addition to locking the doors, you need to restrict access to only those who need it. A good means of doing so is with electronic locks. An additional benefit to using electronic locks is that they are capable of recording information about who opened the locks and when.

In high security areas, the installation of tailgating prevention measures may also be a good idea. Tailgating is when somebody without authorization follows directly behind somebody who is authorized. The use of a turnstile can prevent tailgating.

Badges are another means of implementing an additional layer of physical security. Badges allow you to know who is who. If the badge is configured correctly, you can look at a badge and see who has authorization to be in an area. Electronic locks can actually be activated by badges that have magnetic strips, which can be swiped to unlock doors. Some badges have a radio frequency identification (RFID) chip in them, which can also be used to unlock doors.

Highlights:

● Lock the doors at a minimum, especially the doors of equipment rooms and network cabinets.

● Restrict access to those who need it.

● Locks can be electronic in nature and record information on who opened them.

● For high security areas, put in tailgating prevention measures (e.g., a turnstile).

● Require badges to know who is who and who has the correct authorization.

● Badges can have a magnetic strip or an RFID chip used to unlock doors.

Security at the Node Level

A number of measures can be taken to implement additional security at the node, or PC, level. None of the security measures you implement will make any difference if you don’t first secure the PC.

Privacy Filters

A fairly simple physical security measure that can be implemented at the node level is the use of privacy filters. Such filters prevent shoulder surfing — that is, someone looking over someone else's shoulder to see what's on a screen with the goal of getting sensitive information. Privacy filters restrict the viewing angle at which a screen can be seen, ensuring that sensitive information is kept private.

Employee Badges

Another physical security measure that can be implemented at the node level is the use of badges with a magnetic strip. If this is implemented, readers can be installed on PCs requiring the badges to be inserted before users can log in. This method is especially useful when the login process also requires a PIN. Requiring both the badge and the pin creates a multifactor authentication process.

Biometrics

Biometric measures can be employed to make people prove who they are — not just by what they know but with a physical feature. Fingerprints, retinal patterns, voice patterns, and facial recognition can all be used in an effort to provide additional physical security.

Tokens and Key Fobs

Another security measure that is physical in nature is the use of tokens or key fobs. These can be used both onsite and for remote login. The token or key fob supplies a security code that changes on a regular basis — a rolling code. The changes usually occur every 60 seconds. When logging in, a user looks at the key fob and supplies the code that's on the screen at that time, along with a username and password. The authentication server knows what the code should be at any given time, and the key fob can be set to each individual user for additional security.

Securing Physical Documents and Passwords

As a best practice, important administrative passwords should be documented in the case of an emergency. These passwords need to be secured. At a minimum, they should be in a locked file cabinet with limited access. Access to such a cabinet should only be given to those who have a verified need to be in it.

When documents with sensitive information need to be disposed of, you should employ the use of a cross-cut shredder at the minimum. A better solution is the use of a national document disposal company.

Key Admin Passwords:

● Need to be documented in case of emergency.

● Need to be secured; at a minimum, should be in a locked file cabinet with limited access.

Handling Sensitive Documents:

● Prevent dumpster diving.

● Use a cross-cut shredder at a minimum.

There are many areas where digital security measures should be implemented. Security measures should reside at both the computer level and at the network equipment level. The measures that get enacted will be different for each.

Antivirus

One important area of digital security involves antivirus applications. Antivirus software should be installed, active, and up-to-date on every machine. If it's not installed, it can't stop a virus. If it's not active, it can't stop a virus. If it's not up-to-date, it won't recognize a virus.

Highlights:

● Antivirus software should be installed, active, and up-to-date on every machine.

● Make sure your antivirus software is able to do its job effectively.

Antispyware

Antispyware is closely related to antivirus. As a matter of fact, a lot of antivirus software packages have antispyware functionality built into them. Spyware is malicious code that collects information about a system. It may change some system settings, but what it mainly does is to collect information and then send that information off to a remote site. Antispyware can prevent the malicious code from running. It can also assess a system, help the user determine if spyware is present, and evaluate the level of threat that the spyware represents.

Highlights:

● Spyware is malicious code that collects information about the system and may change some settings.

A discussion on digital security would not be complete without including information on firewalls. Software firewalls should be installed and active on every machine.     PCS 16 /Data Destruction and Disposal

    Overview

    In this module, we will review different methods for destroying data and hard drives. We will also discuss methods to overwrite a hard drive correctly.

    Low-Level Format vs. Standard Format

    Most of us are familiar with formatting disk drives. A common myth is that formatting disk drives destroys data. The primary purpose of formatting a disk is to prepare it for a new operating system installation, and in order for that to be effective, any data currently on the disk needs to be erased or set off by internal pointers that allow the data occupying those sectors to be saved and rewritten after the reformat.

    The standard format process, or high-level format, can be done two ways — the quick format and the full format. The quick format prepares a disk to receive a file structure, such as FAT32 or NTFS, while the full format also checks the drive for bad sectors and marks them. In all versions of Windows since Windows Vista, the full format also writes zeros and ones to the partitions.

    A quick format doesn't really destroy data, it merely removes the index for the file structure, making it slightly harder to find a file. However, the data is not gone, and there are a number of software tools available that will recover files from a disk that has undergone a quick format. There are, however, tools available that will write random zeros and ones to the disk during the formatting process, which is an effective data destruction method.

    Low-level formatting is different, and is an effective means of destroying data, but cannot be truly accomplished after a hard disk drive is manufactured. Due to the nature of modern ATA/IDE drives, where the disk geometry varies by manufacturer, true low-level formatting occurs only during the manufacturing process and cannot be redone after a disk has been placed into service by a user or technician. This is because the low-level format actually defines the position of tracks and sectors on the drive. However, the major HDD manufacturers do make utilities available that will reinitialize a disk, test it for bad sectors, write zeros and ones to the sectors, and repartition their own disks. This is the closest technicians can get to low-level formatting.

    Standard formatting occurs from within the operating system. For example, a technician can connect a new external hard disk in Windows File Explorer and simply right-click the disk and then select Format, as shown in the image below. This will then perform a standard format of the new drive.

    Windows File Explorer pop-up menu with formatting option. Images used under CC-BY license from Jason Dion.

    This process will then erase the file allocation table (FAT) if the drive is formatted with a FAT file system, or erase the file pointers if the file system is NTFS (New Technology File System). This does not erase the data on the drive, only the method to access the data through a normal operating system. It is still possible for a technician to recover the data using software tools built for that purpose.

      Hard Drive Data Destruction Methods

    There are three accepted ways to perform HDD data destruction. They are:

    • Clearing
    • Sanitizing or purging
    • Physical destruction

    If the disk is to be repurposed in another workstation within the company, or the workstation is to be donated to another organization, it is important that the data be destroyed without destroying the hard disk. To accomplish this task, a technician can perform a low-level format using the HDD manufacturer’s software and ensure that it writes zeros and ones to the disk at least one time. This is known as clearing .

    There are many programs on the market that meet or exceed the Department of Defense 5220.22-M standard. This refers to the 1997 Department of Defense National Industrial Security Program Operating Manual (NISPOM) guidelines requiring government contractors to follow exact procedures for dealing with classified data needing to be sanitized . Seven passes of rewrites became the de facto standard adopted in the computer industry as an effective means to recycle HDDs and keep them in service.

    Peter Gutmann increased this standard to a 35-pass overwrite. During this process, each and every bit of the hard disk is overwritten with a zero or one or a random character. Once the end of the disk is reached, the program starts again at the beginning until the entire disk has been overwritten at least 35 times. For most uses, however, seven times is sufficient to prevent data recovery.

    Programs that accomplish this are often referred to as disk wiping, disk scrubbing, or drive wiping programs. Note that these methods take a significant amount of time, especially if the hard disk is large in capacity, since every bit has to be overwritten numerous times.

    Due to the increased awareness of security and changing nature of HDD storage design, the industry is moving away from the DOD 5220.22-M standard, which is no longer in publication. According to an article by Brian Smithson on Infosec Island, the current standard to follow is the National Institute of Standards and Technology “NIST Special Publication 800-88.” Current research is leading the industry to use the “secure erase” techniques identified in this standard, and finds that a “single-pass overwrite” is sufficient.

    Another option to permanently purge the data on a disk is to use electromagnets to degauss (demagnetize) the hard disk. Systems that demagnetize disks are very expensive and are only in use in very high-security environments. Some companies offer disposal of hard disk drives through a pay-for-service model. Often, it is more economical for small businesses to pay for disposal service than to buy expensive equipment.

    When dealing with CDs, DVDs, or floppy disks, the best method of destruction is physical destruction, because these media are very inexpensive to replace. The easiest method to use to destroy these are shredders specifically built for this task, which usually cost around $50–$100.

    While a quick erase of the data can prevent access to it by a normal computer user, the residual data left after a removal process, known as data remanence, can still be retrieved by most technicians or hackers. Data recovery tools are becoming more readily available, better at recovering data, and easier to use, which makes recovery of deleted files well within the reach of many users. For this reason, the best method to make data inaccessible is to physically destroy the hard drive. This can be accomplished via a number of different methods, including destroying the platters using a hammer, drill, or other mechanical device, leaving the drive only usable for recycling as scrap metal. Destruction is the most secure method to permanently eliminate data.

    Choosing the Right Method for Secure Data Destruction

    When time is more of a concern, and data security is less of a concern, a quick format can be appropriate. However, because a skilled technician or hacker can recover data after a quick format, it is best to do a full format when a computer will be repurposed within a company.

    If you or your company is throwing away a computer, there is a concern that sensitive or confidential information might be accessed by another person. Therefore, a quick format is not the best option, and, even though a full format might be sufficient, liability and responsibility for ensuring nobody can ever access any old data may demand physical destruction, such as drilling holes through the hard drive platters, smashing the drives with a hammer, or burning the drives. Physical destruction is the best permanent solution to prevent data recovery.

    If you or your company is selling or transferring the computer to another organization, physical destruction is not an option, and quick formatting is insufficient. In this instance, the disk should be wiped following the DOD 5220.22-M standard by overwriting the disk with zeros and ones in each bit seven to 35 times or using Secure Erase, as discussed in “NIST Special Publication 800-88.”

    人身安全

    PCS 14 /數字安全

    除了物理安全性之外，計算機還必須通過數字安全性得到保護。這採取多種形式，包括使用防病毒/反間諜軟件/反惡意軟件，防火牆，正確的用戶身份驗證，保護無線網絡以及保護網絡的其餘部分。

    防病毒，反間諜軟件和反惡意軟件

    無論您是在商務環境還是家用計算機環境中，每個設備（平板電腦，手機或工作站）都應安裝防病毒軟件，反間諜軟件和反惡意軟件。現在，大多數防病毒軟件都包括針對這三種病毒的掃描程序。這是一種專用軟件，可在啟動時啟動，並在計算機的後台運行，從而實時保護設備免受感染。默認情況下，防病毒軟件將在計算機啟動時對其進行掃描，以確保不存在會影響啟動過程的啟動病毒或rootkit。此外，應安排軟件定期對已知和可疑威脅自動運行全面掃描。

    該軟件必須保持最新狀態，因為始終會創建新的病毒和惡意軟件。除了保持程序更新之外，跟踪最新威脅的“簽名”文件還應每天檢查更新。大多數防病毒程序都是按訂閱運行的，最終用戶必須支付年費才能使軟件保持最新。但是，第三方供應商提供了出色的免費軟件程序。此外，該軟件還提供對病毒/惡意軟件威脅百科全書的訪問，使用戶可以進一步了解存在的特定威脅。

    運行防病毒/防惡意軟件會保護系統免受可能不僅影響該設備而且影響整個網絡的軟件威脅。通常，防病毒軟件是作為產品套件的一部分提供的，其中包括防火牆，瀏覽器保護，密碼管理以及允許基於權限訪問Internet的家族控件。優秀的防病毒軟件還將集成到用戶本地安裝的電子郵件程序中，以確保掃描所有下載的文件以及已發送/已接收的電子郵件。

    防火牆

    防火牆是旨在檢查穿越網絡連接的數據包的硬件設備或軟件程序。這包括IP標頭信息，例如目標或源IP地址，正在使用的應用程序端口和數據或正在使用的協議。根據此信息，防火牆將確定是允許還是阻止特定的網絡流量。在Windows Vista（通過配置）和Windows 7（默認）上，有一種雙向防火牆解決方案。除了在每台計算機上本地運行之外，基於Windows軟件的防火牆還可以由Windows服務器管理。

    硬件防火牆是具有專門設計的操作系統的專用網絡設備，可以更有效地保護網絡中的多個設備。通常，網絡利用網絡邊界處的硬件防火牆來保護整個網絡免受廣泛的攻擊，而軟件防火牆被配置為保護特定的工作站免受特定目標的攻擊。技術人員切勿在工作站上安裝多個軟件防火牆，因為兩個防火牆可能會衝突並阻止有效的網絡流量正常流動。另外，請注意，Windows帶有集成到操作系統中的自己的軟件防火牆，但是如果需要第三方更深入的解決方案，可以將其關閉。

    如果防火牆配置錯誤，則會阻止合法的網絡流量。如果程序似乎無法正常運行，技術人員必須檢查防火牆設置，以確保在防火牆中打開了正確的TCP或UDP端口。例如，如果用戶抱怨其電子郵件程序無法發送電子郵件，則可能是防火牆配置為阻止通過端口25（SMTP-簡單郵件傳輸協議-服務器）的通信。

    PCS 15 /威脅與實踐

    總覽

    在本模塊中，我們將探討不同類型的技術和非技術安全威脅。我們還將討論諸如特洛伊木馬之類的威脅，它們結合了技術和非技術攻擊技術。此外，我們將學習有助於防止攻擊的策略。

    社會工程學

    社會工程是主要的安全威脅。社會工程學是指使用各種技術說服計算機用戶提供對其係統的訪問。計算機和網絡黑客凱文·米特尼克（Kevin Mitnick）對此表示歡迎，他說：“人們傾向於採取捷徑。他們可能知道他們不應該提供某些信息，但是害怕不友善，害怕顯得無知，害怕被認為是權威人物-所有這些都是觸發因素，社會工程師可以使用它們來說服一個人來推翻既定的安全程序。” 例如，如果黑客舉著雙手走到裝扮成包裹運送人的辦公樓的安全區域，您認為有多少人會打開大門讓他進入？這種欺騙是社會工程學的核心。

    預先發短信

    接待員接到電話，呼叫者說：“您好，我叫IT服務台的Bob。我們檢測到您的計算機密碼已被盜用，需要更改。首先，您可以為我驗證您的姓名和生日嗎？” 此時，接待員將提供這些詳細信息，這些詳細信息可用於訪問其他帳戶。隨著呼叫的繼續，攻擊者會詢問接待員的用戶名和密碼，以“驗證”該帳戶已損壞。最後，呼叫者已破壞了接待員的帳戶。

    網絡釣魚

    網絡釣魚是社會工程學的另一種形式。網絡釣魚攻擊是攻擊者獲得系統訪問權限或獲得有關用戶帳戶或財務數據的信息的一種有效且簡便的方法。傳統上，網絡釣魚涉及到最終用戶接收到一封電子郵件，指出他的帳戶已被盜用，或者是另一個類似的前提。在電子郵件中，有一個鏈接可單擊以重置密碼或登錄其帳戶以驗證數據。

    當用戶單擊鏈接時，他將被帶到一個看起來像他的銀行或其他服務的網站，但是地址可能略有錯誤。例如，不是將用戶帶到Facebook.com，而是Facebooks.com。當用戶輸入用戶名和密碼時，該用戶將被轉發到正確的站點，但是攻擊者現在可以在將來的攻擊中使用其受感染的憑據。對於銀行帳戶或經紀帳戶，攻擊者現在可以訪問該帳戶並將帳戶中的資金轉給自己或收集其他詳細信息以利用漏洞利用獲利。

    誘餌

    當攻擊者離開物理媒體（例如CD，DVD或USB拇指驅動器）到處尋找受害者時，就會發生誘餌。一旦受害者找到了媒體，她就會好奇地發現媒體上的內容（了解所有者將歸還誰，或者查看媒體中包含的機密數據）。通常，這些受污染的設備可能會以各種看似合法的方式呈現給受害者，例如免費贈送禮物或作為軟件更新。設備越誘人，例如留在智能手機上，或者受害人獲得設備的場景（例如免費抽獎）越可信，受害人就越難以識別這種情況的可能性。誘餌計劃。防止誘餌攻擊的最佳方法是確保員工意識到自己的存在，因此他們可以抵制誘惑。

    肩衝浪

    當攻擊者試圖通過在用戶的屏幕，鍵盤或鍵盤上查看用戶的“肩膀”來獲取機密信息時，就會發生肩膀衝浪。例如，有人在ATM機上抬頭看著您在鍵盤上輸入PIN時看到。另一個示例可能是，如果您正在查看一家公司的業務員的屏幕。店員可能仍在屏幕上顯示另一個人的帳戶（和財務信息）。一些肩上沖浪者通過使用鏡子，偷窺角落或什至擁有隱藏的攝像機來捕獲他們尋求的詳細信息來暗中行事。應對此類攻擊的最佳防禦方法是了解您的周圍環境，並確保人們與您的終端保持安全的視覺距離。為了防止筆記本電腦和移動設備上的肩膀衝浪，您還可以使用隱私屏幕保護膜。這些特殊的屏幕保護器可防止以一定角度觀看屏幕，從而使除直接注視屏幕的用戶以外的任何人都難以看清顯示屏。

    尾隨和Pi帶

    當攻擊者未經其許可而緊跟在授權人員的後面，試圖獲得對安全設施的訪問時，就會發生尾隨。如果這個訪問是取得了與被授權人的同意，這被稱為捎帶。這方面的一個很好的例子是，當送貨員（如比薩餅送貨員或鮮花送貨員）要求另一名僱員舉門（在授權人員刷了身份證之後），因為送貨員的手太滿了。產品正在交付。這樣，“送貨人”就可以未經授權進入建築物。這種攻擊依賴於大多數人固有的善良，很難教導員工放棄這種普通禮貌。

    PCS 15 /惡意軟件

    社交工程是一種非常有效的途徑，可以在不完全依賴技術領域的情況下訪問系統。除了社會工程學之外，攻擊者還可以通過惡意軟件進行技術攻擊。值得注意的是，將社會工程學與惡意軟件相結合的攻擊可能是一個重大威脅。

    惡意軟件一詞是惡意軟件和軟件一詞的組合，用於描述用於破壞或獲得對計算機或網絡的未經授權訪問的任何軟件。有許多類型的惡意軟件。在本課程中，我們將討論病毒，蠕蟲，特洛伊木馬，rootkit，間諜軟件，遠程訪問，廣告軟件，灰色軟件和垃圾郵件。

    病毒

    病毒是一種軟件程序，可以感染操作系統中的文件，對系統造成嚴重破壞，並且可以執行攻擊者的期望。病毒需要復制才能傳播，並且只有在用戶執行包含病毒的受感染程序時才能完成複制。蠕蟲與病毒相似，但是它們會在網絡中自我複制。蠕蟲是一種非常有效的惡意軟件，因為它們可以自動複製而無需用戶干預。這也使蠕蟲非常危險。特洛伊木馬程序是自稱有用的程序，但實際上在計算機上安裝了有害程序，包括間諜軟件，遠程訪問和rootkit。特洛伊木馬通常與網絡工程（例如網絡釣魚）結合使用，並用作對計算機和網絡造成更嚴重威脅的傳遞機制。

    根套件

    Rootkit是一種隱藏方法，許多類型的惡意軟件都使用Rootkit阻止常規的防病毒和反惡意軟件程序對其進行檢測。Rootkit傾向於覆蓋和重命名系統文件，用它們自己替換這些必需的文件。這將惡意軟件嵌入到操作系統中，從而使得很難通過標準防病毒軟件進行檢測，而是需要常規操作系統之外的高級檢測技術來檢測和刪除這種惡意軟件變體。每次系統啟動時都會激活Rootkit。

    間諜軟體

    間諜軟件是監視系統活動並將網絡搜索或其他活動的詳細信息傳輸到遠程計算機的軟件。如果您在瀏覽Internet時出現多個彈出窗口，則表明間諜軟件已感染您的計算機，這是一個很好的信號。其中一些彈出窗口可能為您提供了刪除病毒或感染的能力，但是您絕對不要使用它們。這些都是騙局，通常是流氓或假冒的防病毒/惡意軟件產品，通常會導致您的計算機和網絡受到更深層次的感染。間諜軟件還會降低計算機的性能，通過，腫的軟件代碼和間諜活動浪費寶貴的計算資源。

    遠程訪問

    遠程訪問惡意軟件程序可對系統進行未經授權的控制，並可用於建立受感染計算機的網絡。這些惡意軟件程序也稱為殭屍網絡。通過遠程訪問，攻擊者可以控制您的計算機，捕獲您的擊鍵，打開網絡攝像頭並將文件複製到自己的計算機上，從而可以訪問您的機密記錄和帳戶。

    廣告軟件，灰色軟件和垃圾郵件

    廣告軟件是顯示與您的網絡搜索和活動有關的彈出廣告和標語的軟件。Grayware是撥號程序，笑話程序，廣告軟件和間諜軟件程序的通用術語。垃圾郵件是來自未知實體的垃圾郵件，非常類似於您家中以物理形式收到的垃圾郵件。垃圾郵件可能是惡意的（例如，試圖向您發送病毒），詐騙（例如，試圖使您匯款），也可能只是荒謬的（例如，向您發送笑話或其他您不想要的消息）。

    垃圾郵件通常是由攻擊者發送的，它通過破解用戶的密碼來劫持用戶的電子郵件帳戶，然後使用該帳戶發送大量垃圾郵件。因此，重要的是經常更改您的電子郵件帳戶密碼，以防止垃圾郵件發送者破解您的密碼並使用您的帳戶。

    PCS 15 /安全意識

    在當今的現代計算機時代，“一盎司預防勝於一磅治療”的格言仍然是正確的。保護您的網絡和防止惡意軟件感染的最有效方法之一是確保您的用戶正在接受適當的網絡安全教育。預防攻擊和感染將為您節省大量時間和資源，否則將需要花費大量時間來排除感染。

    對用戶進行培訓，使他們掌握在工作和家庭中都保持最新的反惡意軟件（例如防病毒和反間諜軟件）的方法。用戶需要了解如何掃描系統中的惡意軟件，以及如何配置其應用程序以進行自動掃描。也需要對各種惡意軟件類型/技術有基本的了解，以便用戶可以識別受感染系統的症狀。每個人都應該掃描可移動媒體以查找惡意軟件，並防止在其係統上發生自動運行，因為這是攻擊者的常見入侵媒介。如果計算機被感染，用戶應了解您組織的策略，即如何應對惡意軟件威脅，如何隔離可疑文件以及如何防止感染蔓延。

    此外，良好的反網絡釣魚教育至關重要，因為這是當今發生的高級持續威脅（APT）最常見的媒介之一。反網絡釣魚教育將有助於防止用戶進行社交工程，以使其不知不覺地打開了攻擊門的大門，例如單擊可疑電子郵件所附的特洛伊木馬。

    安全從門開始。實際上，它始於每一扇門。鎖門是第一安全等級。您建築物的邊界應該是您的第一道防線。部署的安全級別應由所需的安全性決定。隨著對整體安全性的需求增加，物理安全性級別也應隨之增加。

    雖然鎖住每扇門可能不切實際，但某些門絕對應該鎖上。存放服務器的設備或服務器室的門應上鎖。網絡機櫃的門也應該被鎖定，即使它們在服務器機房中也是如此。除了鎖定門外，您還需要限制只有需要它的人才能進入。使用電子鎖是一種很好的方法。使用電子鎖的另一個好處是，它們能夠記錄有關誰打開鎖以及何時打開的信息。

    在安全性高的地區，安裝防拖尾措施也可能是一個好主意。尾隨是指未經授權的人直接跟隨在經過授權的人之後。使用旋轉柵門可以防止產生殘料。

    徽章是實現物理安全性的另一層的另一種方法。徽章可讓您知道誰是誰。如果徽章配置正確，則可以查看徽章並查看誰有權在該區域中。電子鎖實際上可以由帶有磁條的徽章激活，可以滑動磁條來解鎖門。一些徽章上裝有射頻識別（RFID）芯片，也可以用來解鎖門。

    強調：

    ●至少要鎖好門，尤其是機房和網絡機櫃的門。

    ●限制訪問需要它的人。

    ●鎖本質上可以是電子鎖，並且可以記錄誰打開了鎖的信息。

    ●對於安全性較高的區域，請採取防止拖尾的措施（例如，旋轉閘門）。

    ●需要徽章才能知道誰是誰，誰擁有正確的授權。

    ●徽章可以具有用於解鎖門的磁條或RFID芯片。

    隱私過濾器

    可以在節點級別實施的一種相當簡單的物理安全措施是使用隱私篩選器。這樣的過濾器可以防止肩膀衝浪，也就是說，有人看著別人的肩膀看屏幕上的內容，目的是獲取敏感信息。隱私過濾器限制了可以看到屏幕的視角，從而確保敏感信息不公開。

    員工徽章

    可以在節點級別實施的另一項物理安全措施是使用帶有磁條的徽章。如果實現了此功能，則可以將閱讀器安裝在需要在用戶登錄之前插入徽章的PC上。當登錄過程也需要PIN時，此方法特別有用。同時需要徽章和大頭針會創建一個多因素身份驗證過程。

    生物識別

    可以採用生物特徵測量方法來使人們證明自己的身份-不僅通過他們所知道的，而且具有身體特徵。指紋，視網膜圖案，聲音圖案和麵部識別都可以用於提供額外的物理安全性。

    帶弊害密碼卡

    本質上的另一種物理安全措施是使用令牌或密鑰卡。這些可以在現場使用或用於遠程登錄。令牌或密鑰卡提供了定期更改的安全代碼-滾動代碼。這些更改通常每60秒發生一次。登錄時，用戶查看鑰匙扣，並提供當時屏幕上的代碼以及用戶名和密碼。身份驗證服務器知道在任何給定時間應該是什麼代碼，並且可以將密鑰卡設置給每個單獨的用戶以提高安全性。

    低級格式與標準格式

    我們大多數人都熟悉格式化磁盤驅動器。一個普遍的誤解是格式化磁盤驅動器會破壞數據。格式化磁盤的主要目的是為新的操作系統安裝做好準備，並且為了使其生效，磁盤上當前的任何數據都需要通過內部指針擦除或設置，以使數據佔據這些扇區。重新格式化後保存並重寫。

    標準格式過程或高級格式可以通過兩種方式完成-快速格式和完整格式。快速格式可以使磁盤準備好接收文件結構，例如FAT32或NTFS，而完整格式還可以檢查驅動器中的壞扇區並對其進行標記。自Windows Vista以來的所有Windows版本中，完整格式還將零和一寫入分區。

    快速格式並沒有真正破壞數據，它只是刪除了文件結構的索引，這使得查找文件稍微困難一些。但是，數據並沒有消失，並且有許多可用的軟件工具可以從經過快速格式化的磁盤中恢復文件。但是，有一些工具可以在格式化過程中將隨機零和一寫入磁盤，這是一種有效的數據銷毀方法。

    低級格式化是不同的，它是銷毀數據的有效手段，但在製造硬盤驅動器後無法真正完成。由於現代ATA / IDE驅動器的特性，磁盤的幾何形狀因製造商而異，真正的低級格式化僅在製造過程中發生，並且在用戶或技術人員將磁盤投入使用後無法重做。這是因為低級格式實際上定義了驅動器上磁道和扇區的位置。但是，主要的HDD製造商確實提供了實用程序，這些實用程序將重新初始化磁盤，測試其壞扇區，將零和一個寫入扇區以及重新分區自己的磁盤。這是技術人員可以進行的最底層格式化。

    標準格式從操作系統內部發生。例如，技術人員可以在Windows File Explorer中連接新的外部硬盤，然後右鍵單擊該磁盤，然後選擇Format，如下圖所示。然後，它將執行新驅動器的標準格式。

    共有三種執行HDD數據銷毀的方法。他們是：

    • 清算
    • 消毒或清除
    • 物理破壞

    如果要將磁盤重新用於公司內的另一個工作站，或者將該工作站捐贈給另一個組織，則在不破壞硬盤的情況下破壞數據很重要。為了完成此任務，技術人員可以使用HDD製造商的軟件執行低級格式化，並確保其至少一次將零和一寫入磁盤。這稱為 清算。

    市場上有許多符合或超過美國國防部5220.22-M標準的程序。這指的是1997年國防部國家工業安全計劃操作手冊（NISPOM）指南，要求政府承包商遵循確切的程序處理需要清理的機密數據 。七次重寫已成為計算機行業採用的事實上的標準，成為回收HDD並使它們保持使用狀態的有效手段。

    彼得·古特曼（Peter Gutmann） 將該標準提高到35次覆蓋。在此過程中，硬盤的每個位都被零或一個或隨機字符覆蓋。到達磁盤末尾後，程序將從頭開始再次啟動，直到整個磁盤至少被覆蓋35次為止。但是，對於大多數用途而言，七次足以防止數據恢復。

    完成此操作的程序通常稱為磁盤擦除，磁盤清理或驅動器擦除程序。請注意，這些方法要花費大量時間，尤其是在硬盤容量較大時，因為每個位都必須被重寫多次。

    由於人們對安全性的日益提高的意識以及HDD存儲設計的不斷變化的性質，該行業正在遠離已不再公開的DOD 5220.22-M標準。根據布賴恩·史密森（Brian Smithson）在Infosec島上的一篇文章，當前要遵循的標準是美國國家標準技術研究院的“ NIST特別出版物800-88”。當前的研究正在引導業界使用此標準中確定的“安全擦除”技術，並發現“單遍覆蓋”就足夠了。

    永久清除磁盤上數據的另一種方法是使用電磁體對硬盤進行消磁（消磁）。使磁盤消磁的系統非常昂貴，並且僅在非常安全的環境中使用。一些公司通過按服務付費模式提供硬盤驅動器處置。通常，與購買昂貴的設備相比，小型企業為處置服務付費更經濟。

    處理CD，DVD或軟盤時，最好的銷毀方法是物理銷毀，因為這些介質的更換成本非常低廉。銷毀這些物品最簡單的方法是專門為該任務而建造的碎紙機，通常價值約50至100美元。

    儘管快速擦除數據可以防止普通計算機用戶訪問數據，但是大多數技術人員或黑客仍可以檢索到刪除過程後留下的剩餘數據，稱為數據保留。數據恢復工具變得越來越容易使用，更擅長恢復數據，並且更易於使用，這使得恢復已刪除文件的工作在許多用戶的範圍之內。因此，使數據不可訪問的最佳方法是物理破壞硬盤驅動器。這可以通過許多不同的方法來完成，包括使用錘子，鑽頭或其他機械設備破壞盤片，使驅動器僅可作為廢金屬回收利用。銷毀是永久刪除數據的最安全方法。

    選擇正確的方法進行安全的數據銷毀

    當時間更受關注，而數選擇正確的方法進行安全的數據銷毀據安全性不那麼受關注時，可以採用快速格式化。但是，由於熟練的技術人員或黑客可以在快速格式化後恢復數據，因此當公司內部重新使用計算機時，最好進行完整格式化。

    如果您或您的公司丟棄了計算機，則可能會擔心其他人可能訪問敏感或機密信息。因此，快速格式化不是最佳選擇，即使完整格式可能就足夠了，但要確保任何人都無法訪問任何舊數據的責任和責任可能需要物理破壞，例如在硬盤驅動器盤片上鑽孔，粉碎用錘子敲擊驅動器，或燒毀驅動器。物理破壞是防止數據恢復的最佳永久解決方案。

    如果您或您的公司正在出售計算機或將計算機轉讓給另一個組織，則無法進行物理銷毀，並且快速格式化還不夠。在這種情況下，應按照DOD 5220.22-M標準擦除磁盤，方法是用0和1的每一位覆蓋磁盤七到35次，或使用安全擦除，如“ NIST特殊出版物800-88”中所述。

    Concepts

    One review concept that is definitely needed (in addition to the user input that is the core of the exercise) is the need to “split” strings.

    The user will input coordinates in the form “row,col”, which input() will then read in as a string. But we really want the numbers that come out of that string, to know which row and column to place the piece at.

    One approach is to use the idea of strings as lists to extract the row and column numbers. This works great if your row and column numbers are always single digits - the row will always be at index 0 and the column will always be at index 2. But this breaks when the numbers are larger than one digit (I know, not going to happen in tic tac toe, but it’s easy to image extending this to other games).

    Instead, there are two string manipulation functions that will help you:

    1. .split() - Takes a string and returns a list, using the separator as the split criteria. So if you have a string name = "John Doe" and do name_list = name.split(" "), name_list will be ["John", "Doe"]. You can use any separator / split character you want. Just remember, that each of the elements returned back will be a string as well.
    2. .strip() - Takes a string and removes the whitespace on the left and right sides of it. So you have a string name = " Michele ", and you do name = name.strip(), and now name will just be "Michele" - nice and clean.
